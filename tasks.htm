Tasks for Timers and Typesetters

Timers


Things you should always do when timing an episode.
Make sure you know how to do all of this.


• Import styles

You should always import AT LEAST the default style from the previous episode, otherwise you can't check for 3-liners. Usually you should import styles for OP/ED and titles as well.
If the show has a lot of other styles, leave it to the typesetter to decide which ones will be used for any particular episode.
In simple terms you should import styles that will certainly be used, and not import ones you're not sure about.


• Set correct script resolution

This goes together with the first point, otherwise it won't work. Make sure you know how to do this right. Don't ever use "resample resolution" unless explicitly told to. [never seen that happen]


• Time and comment out signs

Always time signs, unless the typesetter tells you that you don't have to for that specific episode [because he does that himself]. If you time and not typeset, comment out the signs.


• Import & shift OP/ED

Import from previous episode, not a random older one. These things tend to change sometimes, so you need to use the most up-to-date version. Get script either from the release or from the previous episode folder on the ftp.
Make sure to shift scene timed OP/ED correctly so that there are no bleeds on any lines.


• Avoid overlapping with scrolling jp text

If there's scrolling japanese text on screen [usually after OP/eyecatch], move the subtitles so that they don't overlap with it, which would make them difficult to read.


• Use fades where needed

When a scene fades in or out and subs start/end there, use \fad with appropriate values.
Most commonly {\fad(500,0)} or {\fad(0,500)}.


• Make chapters

Don't forget about chapters. Follow the instructions in the guide.


• Name files properly

We don't want to see any more scripts named "shana.ass" like some derps are capable of.
ALWAYS include show name AND episode number at least. Until you get used to our system and can do everything without mistakes, avoid overwriting existing files. Instead add -timed to your filename and leave the old one alone.
In fact it would be good if you got the untimed one, opened and saved under new name immediately, and kept the old one just in case something fucks up. Never underestimate the usefulness of having backups. In case of scripts it's only a few KB anyway.
If you upload a script and RHE tells you later to fix something in it, you take the script from the ftp [yes, even if you still have it on your HD], because he may have made changes to it already, keep a backup of it, fix what you were told to fix, and upload with a new name [add -fixed or something] to make sure you don't overwrite something he's working on.
We've seen people overwriting scripts that had 2 hours of work done on them, all gone in a second. Don't do that. Make sure your naming makes sense. Preferably inform the person in question what file exactly you have uploaded.


• Import TS for titles

This may not be necessary but you should be able to do it when asked. This means taking the tags for the title typesets from the previous episode and using them for the current titles. In other words tags and style will be from previous ep, timing and text will be from the current one.
If you're not sure how to do this right, you can copy the whole lines from the previous script and paste them into the current one so that you see both current and previous titles, then copypaste from line to line to get the desired result [then delete the prev. ep. ones].


• Do a 0 frame shift backwards at the completion of timing

Shifting - frame mode - zero, backward, all lines. Make sure you know what you're doing.
This should fix most 0.01 or 0.02s overlaps/gaps.


• Italicize internal dialogue that the editor misses

Whether the editor forgets or it happens because of line splitting, if you notice internal dialogue not italicized, fix it.


Make sure you understand everything and know how to do it. If not, reread the guides or ask.





Typesetters


• Styles

You are responsible for the styles that end up in the final release. If you work on TS separately from the script a timer is working on, and they get merged later, either do the merging yourself or make sure the timer knows which styles to use from where.
Technically the timer may not necessarily know much about styles and could just add the typesets to his script without the styles. It's your responsibility to make sure it doesn't happen.


• Fonts

Obviously you're the one picking the fonts so make sure to upload all the ones you've used. Each episode should have a "fonts" folder. If you use the same fonts for each episode then one folder may be enough for the whole show. If it changes between episodes, make a folder for each episode.


• Overlapping with main dialogue

Making sure typesets don't cover main dialogue is your job. Obviously main dialogue should always be on top. Either check that yourself or make sure the timer does that and knows how to use layers.


As a typesetter you should damn well know how things work and not do something stupid like work in the wrong script resolution.
It is also highly recommended to use avisynth to load video in Aegisub to prevent your typsets from having wrong colors in the release.
Also get the ffms2.dll plugin for avisynth and put it in the plugins folder.





Song Styling


• Make a line for first frame of OP/ED

Not a common practice in Commie, but it should be, as it makes things so much easier.
Do not comment out this line!
If you do, the line will end up at the top of the script in the release and whoever is taking the OP/ED from it the next time will only take it without the first line.
So make a regular line with {First OP frame} or something like that. If the OP/ED is fading in and it's unclear which frame is the first, use a different frame that's clear enough for reference, and make a note like for example {ED frame where first kanji appear}.
